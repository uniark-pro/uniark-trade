# uniark-trade 短线王：安装与使用

> 一个只读的浏览器看盘终端，分 **自选 / Alpha** 两个标签页：**自选**盯主流现货（BTC / ETH / BNB / SOL / DOGE），**Alpha** 盯 Binance Alpha 里 ×4 加成的代币；两边都在 K 线 + MACD 之上叠加**高级别周期的 MACD 投影**与背离信号。复用 `divergence.py` / `indicator.py`，自身不重写任何背离算法。
>
> **只读监控，不下单、不接 API Key、不构成交易建议。**

---

## 它是什么（以及不是什么）

**是**：一个看盘辅助。筛出 ×4 代币 → 看结构（三段背离、动量衰竭、多周期共振）→ 自己判断。

**不是**：不是交易信号源（箭头是结构标记，不是买卖指令）、不是自动交易、不是经过回测的策略。

| 层 | 文件 | 职责 | 是否本项目新写 |
|----|------|------|----------------|
| 数据 | `uniark-trade.py` 内 | 拉 Binance Alpha 行情 | 看板自带 |
| 指标 | `indicator.py` | EMA / MACD | **复用现成库** |
| 背离 | `divergence.py` | 三段背离 + 极值补检 | **复用现成库** |
| 多周期 | `htf_overlay.py` | 高级别 MACD 投影 + 顺势过滤 | **本项目新增，可被他人复用** |
| 界面 | `uniark-trade.py` 内 | Flask + lightweight-charts | 看板自带 |

> 也就是说，本看板既是 `divergence.py` 的一个**使用者**（省去了背离代码化的全部工作），又顺手沉淀出一个新库 `htf_overlay.py` 供别的系统复用——见 [HTF_OVERLAY_ZH](HTF_OVERLAY_ZH.md)。这正是分层设计的红利：加一个新交互，底层库一行不动。

## 安装

```bash
cd ~                              # 或者 cd ~/projects/

# 下载源代码
git clone https://github.com/uniark-pro/alpha-4x.git uniark-trade

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境(提示符前会出现 (venv) 标签)
source venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install flask requests pandas numpy

```

### 文件清单（须同目录）

```
uniark-trade.py   # 看板本体（Flask 服务 + 前端 + 后台采集线程）
indicator.py           # 复用：EMA / MACD
divergence.py          # 复用：背离检测
htf_overlay.py         # 复用：高级别 MACD 投影 + 顺势过滤
storage.py             # 本地 K 线缓存层（SQLite），看板秒开的关键
```

> 看板背离阈值（`MISSED_EXTREME_MIN_BARS`）内置在 `divergence.py`，高级别梯子等内置在 `htf_overlay.py`，看板按默认值调用。
>
> 运行后会自动生成 `data/kline_cache.db`（SQLite 缓存）。可随时删除，重启会重建。

### 启动

```bash
cd ~/uniark-trade
source venv/bin/activate    # 每次新开终端都要重新激活
python uniark-trade.py          # 默认端口 5000
python uniark-trade.py 5001     # 指定端口
```

启动后终端打印两个地址：本机 `http://127.0.0.1:端口`、局域网 `http://<局域网IP>:端口`。手机/平板用**局域网地址**，需和电脑同一 Wi-Fi。

> **不要映射到公网。** 后端在服务器侧请求币安公开行情，家用网络无碍，但本工具没有为公网暴露做任何加固。

### 降级行为

如果 `indicator.py` / `divergence.py` / `htf_overlay.py` 没和本文件同目录，或 pandas/numpy 没装好，启动时会打印：

```
[提示] 未能加载背离模块 ...
```

此时看板**照常显示 K 线，但没有 MACD、背离和高级别叠加**。补齐文件 + 装好依赖即可（也可看 `/api/klines` 返回里的 `divOk` 字段确认）。

如果 `storage.py` 缺失，启动时打印 `[采集] 未加载缓存层 …`，看板仍正常运行，只是退回"每次实时拉币安"（无缓存加速）。补上 `storage.py` 即可。

## 本地缓存 / 秒开（消除出图延迟）

为根除"每次点图都要去币安拉一趟"的延迟，看板内置了一层**本地 SQLite 缓存 + 后台采集线程**：

- **读写分离**：`/api/klines` 不再每次打币安——先读本地库 `data/kline_cache.db`，**命中即零网络往返**（亚毫秒级出图）；未命中/过期才回退实时拉一次并写库。高级别叠加的取数同样走缓存。
- **后台采集线程**：随看板启动的守护线程，常驻把**自选 5 个现货 × 6 个周期**增量刷进缓存（低周期勤刷、高周期懒刷，见下表），所以默认进入的自选页**永远是热的**。
- **Alpha 按需缓存**：×4 代币不常驻采集（省请求、避免 bapi 限频）。某 Alpha 币**首次打开**补拉一次（那一次稍慢），之后读缓存；过期后下次访问再补拉一次。
- **指标仍现算**：缓存只存 K 线；MACD / 背离 / 高级别投影仍在缓存到的 K 线上现算（几百根上现算只要毫秒），所以 `indicator.py` / `divergence.py` / `htf_overlay.py` 一行未改。
- **零新依赖**：`sqlite3` 是 Python 标准库，依赖清单不变。

| 周期 | 自选常驻采集间隔 | 读路径"过期"阈值 |
|------|------------------|------------------|
| 1m | 30s | 150s |
| 5m | 60s | 420s |
| 15m | 120s | 1080s |
| 1h | 300s | 3960s |
| 4h | 600s | 15060s |
| 1d | 1800s | 88260s |

> **新鲜度取舍**：阈值取"周期 + 采集间隔 + 余量"，保证常驻的自选永远命中缓存、不冗余打网。代价是按需的 Alpha 在过期窗口内最多落后约一根 K 线——前端每 30s 自动轮询会很快把它补上，手动刷新亦可。想让 Alpha 也常驻或更激进刷新，调 `uniark-trade.py` 里的采集范围 / `STALENESS` 即可。
>
> 首次启动会预热自选（并发拉一遍），终端打印 `[采集] 预热完成：自选 N 条流已入库`。调试可看 `storage.stats()` 或直接查 `data/kline_cache.db`。

## 界面总览

- **标签页**：列表上方分 **自选 / Alpha** 两个标签（默认进入 **自选**）。
  - **自选**：主流现货 BTC / ETH / BNB / SOL / DOGE，走币安**现货**接口（`api.binance.com`），卡片显示 价 / 成交额 / 24h 涨跌。
  - **Alpha**：×4 加成代币，走 Binance **Alpha 盘口**接口，按盘口成交额降序，卡片保留 倍数 / 链 / 流动性 / 估算剩余。
  - 两边共用同一套 K 线 + MACD + 背离 + 高级别叠加管线；自选拉取失败不影响 Alpha 标签（接口相互独立）。
- **右侧**：选中代币的 K 线主图 + MACD 副面板。
- **手机端**：左右合成单视图——先看列表，点一行进入看盘，左上角 `←` 返回。控件栏分两行：周期按钮一行；"参考高级别MACD"下拉 + "背离标记" + "顺势过滤"并排第二行。

顶栏可切中英文、可手动刷新。列表每 60 秒自动刷新，选中的图每 30 秒自动刷新（不重置缩放）。

> **现货数据源**：自选默认用 `api.binance.com`。若所在网络被地域限制访问不了，改 `uniark-trade.py` 顶部的 `SPOT_BASE` 常量即可（可换 `https://api1.binance.com` 或 `https://data-api.binance.vision` 等镜像）。自选区代币在 `WATCHLIST` 列表里增删。

## 背离标记（本级别）怎么读

勾选"背离标记"显示。**本级别**（当前所选周期）的标记：

| 标记 | 含义 |
|------|------|
| **红 ▼**（柱上方） | 顶背离（看跌） |
| **绿 ▲**（柱下方） | 底背离（看涨） |
| **蓝色**箭头 | 未完成（provisional），末段还可能延伸 |
| `L`+数字 | 层级：`L1` 基础三段，`L2+` 嵌套，**`L0`** 动量极值补检 |
| `+` 后缀 | 多尺度同时衰竭（信号更强） |
| `?` 后缀 | 未完成，当前比值只是快照 |

> 1m + 穷尽层级会刷出很多箭头（深层如 L8、L23 都会冒出来）。把 1m 箭头当**入场细节**看就好，**方向对不对，看下面的高级别叠加**。

## 高级别 MACD 叠加

为解决"只盯低级别、看不见大级别趋势"而做。

**开**：MACD 栏上方下拉 **「参考高级别MACD」**——`·自动`（走梯子）/ `·关` / 显式更高周期。开启后图例显示当前参考周期（如"高级别 15m"）。

**默认梯子（紧邻上一档）**：

| 本级别 | 参考 | | 本级别 | 参考 |
|--------|------|---|--------|------|
| 1m | 5m | | 1h | 4h |
| 5m | 15m | | 4h | 1d |
| 15m | 1h | | 1d | 无（下拉置灰） |

**叠加了什么**（都在 MACD 副面板，挂独立隐藏量纲轴，不会压扁本级别 MACD）：

- **浅色色块（绿/红）** = 高级别 MACD 柱，按正负上色。
- **浅色阶梯线** = 高级别 DIF / DEA。
- 价格栏出现 **琥珀色 ▲▼** = 高级别背离箭头，带周期前缀（如 `15m L2`），与本级别红绿箭头一眼区分。

> **无未来函数**：高级别的值在它那根 K 线**收盘后**才更新；投影到本级别时，每根只用"已收盘"的那根高级别值。最右那一格在当前高级别 K 线收盘前不会变——这是它该有的诚实。**读法看色块颜色（绿=向上 / 红=向下），不要跟本级别零轴比高低。**

**边界**：新币历史不够（如叠 1d 需约 35 天日线）→ 图例显示"高级别 X · 数据不足"，不报错；本级别已是 1d → 下拉置灰。底层实现见 [HTF_OVERLAY_ZH](HTF_OVERLAY_ZH.md)。

## 顺势过滤

勾选"顺势过滤"（默认开，**仅在高级别叠加打开时生效**）：按高级别 MACD 方向，**逐根**隐藏逆势的低级别背离。

| 高级别（如 15m）hist | 本级别信号 | 处理 |
|----------------------|------------|------|
| 绿（>0，向上） | 红 ▼ 卖出 | **隐藏**（逆势噪声） |
| 绿（>0，向上） | 绿 ▲ 买入 | 保留 |
| 红（<0，向下） | 绿 ▲ 买入 | **隐藏**（逆势噪声） |
| 红（<0，向下） | 红 ▼ 卖出 | 保留 |
| 该处无方向 / 数据不足 | 任意 | 保留 |

> **逐根**判断：一个卖出箭头只在它正下方的高级别色块为红时才会留下——箭头永远和脚下的高级别色块一致。关掉它即恢复显示全部本级别箭头，方便核对滤掉了哪些。

## 多周期共振：操作纪律

看到本级别箭头，先瞟一眼高级别叠加，再决定要不要当回事：

| 高级别状态 | 本级别信号 | 解读 |
|------------|------------|------|
| 绿块 / 向上 | 绿 ▲ 底背离 | 顺势底背离，高置信度多头 |
| 红块 / 向下 | 红 ▼ 顶背离 | 顺势顶背离，离场 / 别追多 |
| 红块 / 向下 | 绿 ▲ 底背离 | 逆势抄底，低置信，谨慎 |
| 贴零轴 / 走平 | 任意 | 两级别都没方向，震荡区，低置信 |

简言之：高级别**向上** → 本级别多头信号可重视、空头略过；向下反之；走平则都当参考、别重仓。

> **提醒。** 多周期共振、顺大级别趋势是标准的技术分析纪律，本看板把"看不见大级别"这个盲点补上；但它是结构辅助、**不是盈利保证**，仓位与止损仍须自定。

## 看图标准流程

1. **选币**：左侧列表，按成交额从上往下挑。
2. **选周期**：点入场周期（如 1m / 5m）。
3. **确认叠加**：下拉为"参考高级别MACD·自动"，图例出现"高级别 X"。
4. **先看高级别**：MACD 栏阶梯线 + 色块——绿/向上、红/向下、还是走平？
5. **再看本级别箭头**：绿 ▲ / 红 ▼，什么层级。
6. **共振判断**：对照上表，本级别信号与高级别方向是否同向。
7. **风控自定**：仓位、止损、是否进场——工具只给结构。

## 调参速查

改完重启即可生效：

| 想调什么 | 改哪里 | 常量 | 默认 |
|----------|--------|------|------|
| 本级别背离层级深度 | `uniark-trade.py` | `DIV_MAX_LEVEL` | `None`（穷尽） |
| 本级别短段合并阈值 | `uniark-trade.py` | `DIV_MIN_BARS` | `0` |
| 力度衰竭阈值 | `uniark-trade.py` | `DIV_RATIO_THRESHOLD` | `0.5` |
| MACD 预热最少根数 | `uniark-trade.py` | `MIN_BARS_FOR_MACD` | `35` |
| 涨跌配色 | `uniark-trade.py`（JS） | `RED_UP` | `false`（绿涨红跌） |
| 高级别默认梯子 | `htf_overlay.py` | `HTF_LADDER` | 紧邻上一档 |
| 高级别拉取根数 | `htf_overlay.py` | `DEFAULT_FETCH_LIMIT` | `300` |
| L0 极值过短过滤 | `divergence.py` | `MISSED_EXTREME_MIN_BARS` | `9` |
| 自选常驻采集间隔 | `uniark-trade.py` | `COLLECT_CADENCE` | 见缓存章节 |
| 缓存"过期"阈值 | `uniark-trade.py` | `STALENESS` | 见缓存章节 |
| 每流缓存深度 | `uniark-trade.py` | `CACHE_FETCH_LIMIT` | `1000` |

> **箭头太密**：把 `DIV_MAX_LEVEL` 改成 `2`（只看 L1/L2），或调大 `DIV_MIN_BARS`。只影响**本级别**密度，不影响高级别叠加。

## 常见问题

| 现象 | 排查 |
|------|------|
| 只有 K 线，没 MACD / 背离 / 叠加 | 三个库文件是否同目录、pandas/numpy 是否装好；看启动日志 `[提示] 未能加载背离模块` 或 `/api/klines` 的 `divOk` |
| 叠加显示"数据不足" | 该币在所选高级别上历史 < 35 根（新币常见，尤其叠 1d）。换更低参考周期，或等历史积累 |
| 1d 周期下拉灰掉 | 正常，1d 之上无更高周期可投影 |
| 手机连不上 | 用启动打印的**局域网地址**，且与电脑同一 Wi-Fi |
| 想看某币原始盘口 | 浏览器开 `/api/ticker?symbol=ALPHA_162USDT` 看 ticker JSON |
| 自选出图秒开、Alpha 首开略慢 | 正常：自选常驻采集已预热，Alpha 是按需缓存（首开补拉一次，之后秒开） |
| 缓存想清空重建 | 停服删掉 `data/kline_cache.db` 再启动即可；启动日志 `[采集] 未加载缓存层` 则是 `storage.py` 缺失 |
